# 🧠 MiMo-TUI 多轮记忆系统

> 为 MiMo V2.5 Pro 量身打造的长期记忆系统，让 AI 助手能够记住你的偏好、习惯和重要信息。

---

## 概述

MiMo-TUI 的多轮记忆系统允许 AI 助手在**跨会话**中记住重要信息，包括你的编码偏好、项目约定、常用工具选择等。记忆内容会在每次对话开始时自动注入到系统提示中，使 MiMo 能够基于你的历史偏好提供个性化回答。

### 核心特性

| 特性 | 说明 |
|------|------|
| **长期记忆** | 持久化存储，跨会话保留 |
| **自动注入** | 每次对话自动将记忆注入系统提示 |
| **三种写入方式** | LLM 工具调用、命令行、快捷输入 |
| **BM25 档案检索** | 搜索历史对话中的相关内容 |
| **记忆整理** | LLM 辅助去重和整理记忆条目 |
| **上下文压缩优化** | 1M 上下文窗口下优先保留记忆 |

## 快速开始

### 1. 启用记忆功能

```bash
# 设置环境变量启用记忆
export MIMO_MEMORY=on

# 启动 CLI
cargo run --bin mimo
```

### 2. 添加记忆

**方式一：快捷输入（推荐）**

```
[You]> #prefer pytest over unittest
✓ Memory added: "prefer pytest over unittest"
```

以 `#` 开头的单行内容会直接添加到记忆文件，不会触发 API 调用。

**方式二：命令行**

```
[You]> /memory add 这个项目使用 4 空格缩进
✓ Memory added: "这个项目使用 4 空格缩进"
```

**方式三：让 MiMo 自动记住**

你可以直接告诉 MiMo 要记住什么：

```
[You]> 请记住，我喜欢用 rust 而不是 c++ 写系统工具
```

MiMo 会调用 `remember` 工具将这条偏好保存到长期记忆中。

### 3. 查看记忆

```
[You]> /memory

Memory file: /Users/jerrywu/.mimo/memory.md
Entries: 3

- (2026-05-16 10:30 UTC) prefer pytest over unittest
- (2026-05-16 10:31 UTC) 这个项目使用 4 空格缩进
- (2026-05-16 10:32 UTC) 我喜欢用 rust 而不是 c++ 写系统工具
```

## 记忆命令

| 命令 | 功能 |
|------|------|
| `/memory` 或 `/memory show` | 显示当前所有记忆条目 |
| `/memory add <内容>` | 添加一条新的记忆 |
| `/memory clear` | 清空所有记忆 |
| `/memory path` | 显示记忆文件路径 |
| `/memory help` | 显示记忆帮助信息 |

### 快捷输入语法

```
#<笔记内容>     添加记忆（单行，不触发对话）
##<内容>        普通对话（不会被当作记忆）
#<多行          普通对话（含换行不会被当作记忆）
```

## 记忆存储

### 文件位置

```
~/.mimo/memory.md
```

### 文件格式

记忆文件是纯 Markdown 格式，每条记忆是一个带时间戳的列表项：

```markdown
- (2026-05-16 10:30 UTC) prefer pytest over unittest
- (2026-05-16 10:31 UTC) 这个项目使用 4 空格缩进
- (2026-05-16 10:32 UTC) 我喜欢用 rust 而不是 c++ 写系统工具
```

### 大小限制

- **最大文件大小**：100 KiB
- 超过限制时，记忆内容会被截断并显示 `(truncated)` 标记
- 建议保持记忆条目精炼、简洁

### 环境变量

| 变量 | 说明 |
|------|------|
| `MIMO_MEMORY=on` | 启用记忆功能（默认关闭） |

## LLM 可调用的记忆工具

### 1. `remember` - 记住信息

MiMo 可以主动调用此工具保存重要信息到你的记忆文件。

**参数：**
- `note`（必需）：要记住的单句笔记

**示例调用：**
```json
{
  "name": "remember",
  "arguments": {
    "note": "用户偏好使用 dark mode 主题"
  }
}
```

**使用场景：**
- 用户明确表达偏好时
- 发现重复出现的工作模式时
- 项目特定的约定和规范

**不适合存储的内容：**
- 密钥或敏感信息
- 临时任务状态
- 一次性对话片段

### 2. `recall_archive` - 搜索历史对话

使用 BM25 算法搜索之前的对话档案，找回过去的讨论和决策。

**参数：**
- `query`（必需）：搜索关键词
- `max_results`（可选）：返回结果数量，默认 3，最大 10
- `session_id`（可选）：限定搜索特定会话
- `session_content`（可选）：要搜索的会话内容

**返回结果：**
```
[Session: abc-123] (score: 4.52)
...我们讨论了使用 tokio 作为异步运行时...
---
```

**使用场景：**
- 忘记之前做出的技术决策
- 需要回顾过去的讨论细节
- 查找之前提到的代码片段

### 3. `memory_organize` - 整理记忆

当记忆文件变得杂乱或包含重复条目时，MiMo 可以调用此工具重新整理记忆。

**参数：**
- `action`（必需）：`deduplicate` 或 `clean`
- `entries`（必需）：整理后的记忆条目列表

**使用场景：**
- 记忆中存在重复或矛盾的内容
- 需要将多条相关记忆合并为一条
- 清理过时或不再适用的信息

## 记忆注入机制

### 系统提示注入

每次向 MiMo 发送请求时，系统会自动将记忆内容包装在 XML 标签中并注入到消息开头：

```xml
<user_memory source="/Users/jerrywu/.mimo/memory.md">
- (2026-05-16 10:30 UTC) prefer pytest over unittest
- (2026-05-16 10:31 UTC) 这个项目使用 4 空格缩进
</user_memory>
```

### 注入位置

记忆块被放置在**系统提示的挥发性内容边界之上**，这意味着：
- 记忆内容在每次对话中保持稳定
- MiMo 能够优先关注这些持久化信息
- 不会被上下文压缩影响

---

## 上下文压缩与记忆保护

MiMo V2.5 Pro 拥有 **1,000,000 token** 的超大上下文窗口，记忆系统在压缩策略中做了专门优化：

| 配置 | 值 | 说明 |
|------|------|------|
| 上下文窗口 | 1,000,000 tokens | MiMo V2.5 Pro 原生支持 |
| 压缩触发阈值 | 80% | 上下文使用达到 80% 时触发压缩 |
| 记忆保留比例 | 10% | 始终保留 10% 上下文预算给记忆内容 |
| 压缩目标 | 50% | 压缩后将上下文降至 50% |

当上下文接近阈值时：
1. 系统优先保留记忆内容
2. 压缩最早的对话历史
3. 保留最近 30% 的对话轮次

## 最佳实践

### 适合记忆的 ✅

- 编码偏好（语言、框架、工具选择）
- 项目约定（命名规范、缩进风格）
- 架构决策（为什么选择 A 而不是 B）
- 重要的人名、术语、领域知识
- 重复出现的工作模式

### 不适合记忆的 ❌

- API 密钥、密码等敏感信息
- 临时任务状态（"正在修复 bug #123"）
- 一次性对话内容
- 大量代码片段（应引用文件路径）
- 可能过时的临时信息

### 记忆整理建议

当记忆条目增多后，可以要求 MiMo 进行整理：

```
[You]> 帮我整理一下记忆，去掉重复的，合并相关的条目
```

MiMo 会调用 `memory_organize` 工具，输出整理后的记忆列表供你确认。

## 故障排查

### 记忆功能未启用

```
✗ Memory is disabled. Set MIMO_MEMORY=on to enable.
```

**解决方案：** 确保设置了环境变量 `MIMO_MEMORY=on`

### 记忆文件为空

```
Memory file is empty. Use /memory add <note> to add entries.
```

**解决方案：** 使用 `/memory add` 命令或 `#` 快捷输入添加记忆

### 记忆被截断

当记忆文件超过 100 KiB 时，注入时会被截断。

**解决方案：** 使用 `/memory` 查看完整内容，然后让 MiMo 整理记忆：
```
[You]> 帮我整理一下记忆，去掉重复和过时的条目
```

# 任务 14: 配置管理与环境变量

> 先阅读 `00-项目总览.md` 了解项目背景，然后执行本任务。

## 要求

### 1. 环境变量支持

| 变量 | 说明 | 示例 |
|------|------|------|
| `MIMO_API_KEY` | API 密钥 | `export MIMO_API_KEY=sk-xxx` |
| `MIMO_MEMORY` | 启用记忆（设为 `on`） | `export MIMO_MEMORY=on` |
| `MIMO_MODEL` | 默认模型 | `export MIMO_MODEL=mimo-v2.5-pro` |
| `MIMO_DEBUG` | 调试模式（显示更多日志） | `export MIMO_DEBUG=on` |

### 2. 配置文件（可选，未来扩展）

```toml
# ~/.mimo/config.toml
[general]
model = "mimo-v2.5-pro"
temperature = 0.7
max_tokens = 4096

[memory]
enabled = true

[ui]
theme = "dark"
show_thinking = true
scroll_lines = 3
```

### 3. 配置加载优先级

```
环境变量 > 配置文件 > 默认值
```

### 4. 实现方式

```rust
pub struct AppConfig {
    pub api_key: String,
    pub model: String,
    pub memory_enabled: bool,
    pub debug: bool,
    pub temperature: f64,
    pub max_tokens: usize,
}

impl AppConfig {
    pub fn load() -> Result<Self> {
        // 1. 读取环境变量
        // 2. 尝试读取 ~/.mimo/config.toml
        // 3. 合并并应用默认值
    }

    pub fn get_api_key() -> Result<String> {
        // 优先 MIMO_API_KEY
        // 其次 ~/.mimo/api_key 文件
        // 最后返回错误提示
    }
}
```

### 5. API 密钥加载策略

```
1. 检查 MIMO_API_KEY 环境变量
2. 检查 MIMO_API_KEY_FILE 环境变量（指向文件路径）
3. 读取 ~/.mimo/api_key 文件
4. 如果都未找到，显示设置提示并返回错误
```

## 交付物

- 创建 `packages/apps/tui/src/config.rs`（或利用 `crates/config`）
- 在启动时加载配置
- `cargo check` 编译通过

# 任务 15: 错误处理与用户反馈

> 先阅读 `00-项目总览.md` 了解项目背景，然后执行本任务。

## 要求

### 1. API 错误分类

| 错误类型 | 状态码/特征 | 用户提示 | 建议操作 |
|----------|-------------|----------|----------|
| API 密钥无效 | 401 | "API 密钥无效，请检查 MIMO_API_KEY" | 提示用户重新设置环境变量 |
| 速率限制 | 429 | "请求过于频繁，请稍后重试" | 显示倒计时，自动重试 |
| 服务器错误 | 5xx | "服务器暂时不可用，请稍后重试" | 提供重试按钮 |
| 模型不存在 | 404 | "模型不存在，请检查模型名称" | 提示可用模型列表 |

### 2. 网络错误

| 错误类型 | 用户提示 | 建议操作 |
|----------|----------|----------|
| 连接超时 | "连接超时，请检查网络" | 重试按钮 |
| DNS 解析失败 | "无法连接到服务器" | 检查网络设置 |
| SSL 证书错误 | "证书验证失败" | 检查系统时间/证书 |

### 3. 工具执行错误

| 错误类型 | 用户提示 |
|----------|----------|
| 文件不存在 | "文件不存在: path/to/file" |
| 命令失败 | 显示 stdout/stderr 输出 |
| 权限不足 | "权限不足，请检查文件权限" |

### 4. 用户反馈系统

所有操作都有视觉反馈：

```
类型       颜色    图标    示例
────────   ──────  ─────   ──────────────────────────────
Success    绿色    ✓       ✓ Memory added: "prefer pytest"
Info       蓝色    ℹ       ℹ Connecting to MiMo API...
Warning    黄色    ⚠       ⚠ Context approaching limit
Error      红色    ✗       ✗ API Error: rate limit exceeded
```

### 5. 错误展示组件

```rust
pub enum ErrorDisplay {
    StatusBar,      // 在状态栏显示简短错误
    Toast,          // 弹出 Toast，自动消失
    Dialog,         // 模态对话框，需要用户确认
    Inline,         // 内联显示在消息区
}

pub struct ErrorReporter {
    // ...
}

impl ErrorReporter {
    pub fn report(&self, error: &Error, display: ErrorDisplay);
    pub fn report_api_error(&self, status: u16, message: &str);
    pub fn report_network_error(&self, error: &str);
    pub fn report_tool_error(&self, tool_name: &str, error: &str);
}
```

### 6. 长时间操作进度指示

- LLM 响应中：显示 `◐ 正在思考...` 并闪烁
- 工具执行中：显示 `⚙ 执行中...` 和工具名
- 文件加载中：显示加载进度（如果可能）

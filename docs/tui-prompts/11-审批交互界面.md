# 任务 11: 审批交互界面

> 先阅读 `00-项目总览.md` 了解项目背景，然后执行本任务。

## 要求

当工具调用需要用户审批时，弹出审批对话框。

### 布局

```
┌── Tool Approval ─────────────────────────────────────┐
│                                                       │
│  Tool: shell                                          │
│  Arguments:                                           │
│    command: "rm -rf /tmp/old-build"                   │
│                                                       │
│  Risk: 🔴 High (destructive command)                  │
│                                                       │
│  [a] Approve  [r] Reject  [A] Approve All  [q] Cancel│
│                                                       │
└───────────────────────────────────────────────────────┘
```

### 功能

- 显示工具名称和参数
- 风险评估（低/中/高）：
  - 🔴 High：破坏性命令（rm、sudo 等）
  - 🟡 Medium：文件写入、网络请求
  - 🟢 Low：只读操作（ls、cat、grep 等）
- 审批选项：批准、拒绝、全部批准、取消
- 超时自动拒绝（可配置，默认 60 秒）

### 审批结果

```rust
pub enum ApprovalResult {
    Approved,
    Rejected,
    ApprovedAll,   // 后续工具调用自动批准
    Cancelled,
    Timeout,
}
```

## 交付物

- 创建 `packages/apps/tui/src/components/approval.rs`：审批对话框
- 集成到工具调用流程中
- `cargo check` 编译通过
